# Marketplace RU — Ozon Seller API

Skill для управления продажами на маркетплейсе Ozon через AI-ассистента.

## Описание

Этот skill позволяет OpenClaw агенту управлять магазином на Ozon:
- Мониторить новые заказы и дедлайны отгрузки
- Обновлять цены товаров с контролем изменений
- Управлять остатками на складах
- Получать аналитику и алерты

## Требования

- Учётная запись Ozon Seller
- API ключи (Client-Id + Api-Key) с правами Admin или Content+Analytics
- Настроенные credentials (`mp-setup`)

## Команды

### 🛠️ Настройка

#### `mp-setup`
Интерактивная настройка API credentials.

**Когда использовать:**
- Первый запуск skill
- Обновление API ключей
- Ошибка авторизации (401)

**Пример:**
```
Пользователь: "Настрой подключение к Ozon"
Агент: Выполняю mp-setup...
```

#### `mp-test`
Проверка подключения к Ozon API.

**Когда использовать:**
- После настройки credentials
- Диагностика проблем с API
- Проверка что credentials актуальны

**Пример:**
```
Пользователь: "Проверь подключение к Ozon"
Агент: mp-test → ✅ Ozon API: Подключено
```

### 📦 Заказы

#### `mp-orders list`
Показать новые и активные заказы.

**Опции:**
- `--status <status>` — фильтр по статусу (awaiting_packaging, delivering, delivered)
- `--limit <num>` — количество заказов (по умолчанию 50)
- `--mock` — тестирование без реального API

**Когда использовать:**
- Утренний дайджест
- Проверка новых заказов
- Мониторинг дедлайнов отгрузки

**Примеры:**
```
Пользователь: "Покажи новые заказы на Ozon"
Агент: mp-orders list
Вывод: 📦 Найдено заказов: 3
       Заказ №12345678-0001-1
       Статус: awaiting_packaging
       Отгрузка до: завтра в 12:00

Пользователь: "Какие заказы нужно собрать сегодня?"
Агент: mp-orders list → фильтрует по shipment_date сегодня
```

#### `mp-orders status <order_id>`
Показать детальный статус заказа.

**Параметры:**
- `order_id` — номер отправления (posting_number)

**Примеры:**
```
Пользователь: "Статус заказа 12345678-0001-1"
Агент: mp-orders status 12345678-0001-1
```

#### `mp-orders stats`
Статистика заказов (количество по статусам).

**Примеры:**
```
Пользователь: "Сколько заказов в работе?"
Агент: mp-orders stats
```

### 💰 Цены

#### `mp-prices get <sku>`
Получить текущую цену товара.

**Параметры:**
- `sku` — SKU или offer_id товара

**Примеры:**
```
Пользователь: "Какая цена на товар TWS-PRO-001?"
Агент: mp-prices get TWS-PRO-001
Вывод: Текущая цена: 1999 ₽
       Старая цена: 2499 ₽
```

#### `mp-prices update <sku> <price>`
Обновить цену товара.

**Параметры:**
- `sku` — SKU или offer_id товара
- `price` — новая цена в рублях

**Опции:**
- `--old-price <price>` — установить зачёркнутую цену
- `--yes` — пропустить confirmation (осторожно!)

**⚠️ ВАЖНО:**
- **ВСЕГДА спрашивай подтверждение** перед обновлением цены
- Показывай старую → новую цену и % изменения
- Предупреждай если изменение больше ±50%
- Никогда не используй `--yes` без явного разрешения пользователя

**Примеры:**
```
Пользователь: "Подними цену на TWS-PRO-001 до 2500 рублей"
Агент: 
  1. mp-prices get TWS-PRO-001 (получить текущую цену)
  2. Показать: "Изменить цену с 1999 → 2500 (↑25.06%)?"
  3. Спросить подтверждение
  4. Если подтверждено: mp-prices update TWS-PRO-001 2500

Пользователь: "Поставь цену 500 рублей на товар который стоит 2000"
Агент: "⚠️ Слишком большое снижение (-75%)! Уверены?"
       Только после подтверждения выполнить
```

#### `mp-prices list`
Показать цены всех товаров.

**Опции:**
- `--limit <num>` — количество товаров (по умолчанию 100)

**Примеры:**
```
Пользователь: "Покажи все цены товаров"
Агент: mp-prices list
```

### 📊 Остатки

#### `mp-stocks list`
Показать остатки всех товаров.

**Опции:**
- `--low` — только товары с низкими остатками (< 10 шт)
- `--threshold <num>` — порог низкого остатка (по умолчанию 10)
- `--limit <num>` — количество товаров (по умолчанию 100)

**Примеры:**
```
Пользователь: "Покажи все остатки"
Агент: mp-stocks list

Пользователь: "Какие товары заканчиваются?"
Агент: mp-stocks list --low
Вывод: ⚠️ Товары с низкими остатками:
       • Умная колонка Home Mini: 8 шт
       • Фитнес-браслет: 3 шт

Пользователь: "Что почти закончилось? (меньше 5 штук)"
Агент: mp-stocks list --low --threshold 5
```

#### `mp-stocks get <sku>`
Получить остаток конкретного товара.

**Параметры:**
- `sku` — SKU или offer_id товара

**Примеры:**
```
Пользователь: "Сколько осталось TWS-PRO-001?"
Агент: mp-stocks get TWS-PRO-001
Вывод: Остаток: 45 шт
       Зарезервировано: 5 шт
```

#### `mp-stocks update <sku> <quantity>`
Обновить остаток товара.

**Параметры:**
- `sku` — SKU или offer_id товара
- `quantity` — новое количество (целое число)

**Примеры:**
```
Пользователь: "Установи остаток TWS-PRO-001 на 100 штук"
Агент:
  1. Показать текущий и новый остаток
  2. Спросить подтверждение
  3. mp-stocks update TWS-PRO-001 100
```

## Ограничения и Rate Limits

- **Rate limits:** 100-300 запросов/минуту (зависит от endpoint)
- **Массовое обновление цен:** до 1000 товаров за раз
- **Массовое обновление остатков:** до 100 товаров за раз
- **Заказы FBS:** нужно отгрузить до `shipment_date`

## Автоматизация и use cases

### 1. Утренний дайджест

**Триггер:** Heartbeat в 09:00

**Действия:**
1. `mp-orders list` — новые заказы за ночь
2. `mp-stocks list --low` — товары с низкими остатками
3. Форматировать сводку и отправить пользователю

**Пример:**
```
📊 Утренний отчёт Ozon (16.02.2026)

📦 Новые заказы: 5
   • 3 нужно собрать сегодня
   • 2 можно собрать завтра

⚠️ Низкие остатки:
   • Умная колонка: 8 шт
   • Фитнес-браслет: 3 шт
   → Рекомендую пополнить!
```

### 2. Мониторинг дедлайнов

**Триггер:** Heartbeat каждые 2 часа

**Действия:**
1. `mp-orders list`
2. Проверить `shipment_date`
3. Если дедлайн < 4 часа → алерт пользователю

**Пример:**
```
⚠️ СРОЧНО: Заказ 12345678-0001-1
   Отгрузка через 3 часа!
   Товары: Беспроводные наушники (2 шт)
```

### 3. Умное управление ценами

**Сценарий:** Пользователь хочет поднять цены на 10%

**Действия:**
1. `mp-prices list` — получить все цены
2. Вычислить новые цены (+10%)
3. Показать таблицу изменений
4. Попросить подтверждение для каждого товара
5. Обновить цены по одному

**Важно:** Не обновлять все цены сразу без подтверждения!

### 4. Контроль остатков

**Триггер:** Heartbeat 2 раза в день (утро/вечер)

**Действия:**
1. `mp-stocks list --low --threshold 5`
2. Если найдены товары с остатком < 5 → алерт
3. Предложить пополнить

### 5. Анализ заказов

**Сценарий:** Пользователь спрашивает "Что продаётся лучше всего?"

**Действия:**
1. `mp-orders list --limit 100`
2. Парсить JSON и подсчитать топ-товары по количеству заказов
3. Показать топ-5

## Форматирование вывода

### Для пользователя

Используй человеко-понятный формат:
- ✅ "Заказ №12345678-0001-1"
- ✅ "Отгрузка до: завтра в 12:00"
- ✅ "Цена: 1999 ₽"
- ❌ НЕ: "2026-02-17T12:00:00Z"
- ❌ НЕ: raw JSON

### Эмодзи для статусов

- 📦 Заказы
- 💰 Цены
- 📊 Остатки/аналитика
- ⚠️ Предупреждения
- ✅ Успех
- ❌ Ошибка
- 🔄 В процессе
- ⏰ Дедлайн

## Обработка ошибок

### 401 — Invalid credentials
```
"Ошибка авторизации. Проверьте API ключи.
Выполните: mp-setup"
```

### 403 — Access denied
```
"Доступ запрещён. API ключ не имеет нужных прав.
Создайте новый ключ с правами Admin или Content+Analytics."
```

### 429 — Rate limit
```
"Превышен лимит запросов. Подождите 60 секунд и повторите."
```

### 500/502/503 — API error
```
"Ozon API временно недоступен. Повторите через несколько минут."
```
